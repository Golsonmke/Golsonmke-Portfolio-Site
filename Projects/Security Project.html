Featured Project

Enhanced Information Security with Microsoft Azure Sentinel

Overview

This project demonstrates how I used Microsoft Azure Sentinel (SIEM) with a Windows VM honeypot and an Apache2 web server to collect, analyze, and visualize real-world cyberattack data. The honeypot was intentionally exposed to the internet to attract attacks, and Sentinel was used to ingest logs, enrich them with geolocation data, and display the results on a world map.

Objectives

  • Gain hands-on experience with Azure Sentinel, a leading SIEM tool
  • Deploy a vulnerable VM honeypot to attract attacks
  • Collect and analyze logs of failed RDP login attempts
  • Enrich log data with IP geolocation information using PowerShell and an API
  • Visualize attacker origins and trends on an interactive map in Sentinel

Implementation

  1. Azure Setup: Created an Azure account, deployed a Windows VM honeypot (RDP exposed, firewall disabled), and configured inbound rules to allow global traffic.
  2. Log Collection & Analysis: Configured a Log Analytics Workspace (LAW), built a PowerShell script to extract Event ID 4625 (failed RDP logins), and integrated the IP Geolocation API.
  3. Visualization: Used Sentinel’s dashboard to map global attack sources, observing brute-force login attempts from multiple countries.
  4. Additional Testing: Deployed an Ubuntu VM with Apache2, tested open ports with nmap, and ran nikto scans for vulnerabilities.

Results & Insights

  • Captured real-world global brute-force RDP attacks within hours of exposure
  • Visualized threat origins geographically in Azure Sentinel
  • Demonstrated how quickly attackers find open services on the internet
  • Strengthened skills in SIEM setup, log analysis, and SOC-style workflows

Conclusion

This project reinforced my skills in cloud security, SIEM configuration, log analysis, and threat visualization. It highlighted the importance of proactive monitoring and how quickly unprotected systems are targeted in the wild. Future improvements include refining detection rules and expanding threat intelligence enrichment for more advanced use cases.
